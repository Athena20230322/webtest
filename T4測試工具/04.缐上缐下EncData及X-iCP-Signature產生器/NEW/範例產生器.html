API 説明

API 通用規則

  • 交易傳輸規格
    平台與特店業者間的所有交易一律採用 HTTP 1.1 規範進行電文交換。
  • 文字編碼（編碼方式）
    平台與特店業者間的所有交易電文一律採用 UTF-8 編碼
  • ICP會員驗證逾時 Timeout 時間 業者發動自行決定超時間，但不應少 於 20 秒。
  • 交易逾時 Timeout 時間
    特店業者發動自行決定超時時間，但不應少 10 秒。
  • 特店業者ICP會員驗證金鑰
    Request 需帶上由存取權杖中心所配發的 Ocp-Apim-Subscription-Key，用以判別請求來源端
    是否合法
  • 特店業者金鑰
    一共會有 3 把金鑰，特店業者的 RSA2048 private key，ICP 的 RSA2048 public key，與雙方共同使用對電文加解密的 AES 256 key(MODE:CBC)。
    當特店業者要申請金鑰交換或要更換金鑰時，先向 ICP 取得加密公鑰，特店業者自己產生公私鑰後再與 ICP 交換公鑰，雙方有公私鑰後，即可向 ICP 取得加密 AES 金鑰。
    特店業者傳利用 API 電文時會使用。
  • API 電文：
    有 Header 與 EncData 二部份，Header包括使用的傳送者代號，簽章與加解密的keyID， EncData 必須使用 JSON 格式，再進行加密，加密後的資料再用 base64 編碼。 接收者收到資料後需先對資料先進行 base64 解碼，再進行 AES 解密，以特店業者 RSA 私鑰進行 SHA256 驗簽章是否正確等資料驗證工作，再對電文內容進行處理。
  • 注意：
    1. Header Content-Type 採用 application/x-www-form-urlencoded
    2. Request 電文之加簽用 UTF8 轉 Byte 陣列，資料內容為 EncData
    3. Response 電文之驗簽用 UTF8 轉 Byte 陣列，資料內容為 Body

CHK001-身份驗證請求

特店有ICP會員身份驗證請求時，將呼叫此API進行ICP會員身份驗證 。

  • 介接路徑：
    UAT 環境： https://icash-apim.icashsys.com.tw/member-uat/v1/ICPMemberAuth
    正式環境： https://icash-apim.icashsys.com.tw/member/v1/ICPMemberAuth
  • Request參數說明：
    Header
    No KEY KEY中文 型別 必填 說明
    01 Ocp-Apim-Subscription-Key 存取權杖 String 是 由存取權杖中心配發，用以識別來源端是否合法
    Request Header Example

    {

    "Ocp-Apim-Subscription-Key" : "5f8b04501aa24e07b5adf96ed4b58203"

    }
    Body
    No KEY KEY中文 型別 必填 說明
    01 AuthType 驗證方式 Int 是 1:App推撥驗證
    2:掃碼驗證
    3:簡訊 OTP 驗證
    02 ID 身份証字號 String(20) 是 被驗證者的身份証字號
    (國內:身份証字號/國外:居留證號)
    03 Birthday 出生年月日 String(8) 是 被驗證者的出生年月日(EX:19821108)
    04 ExpireMinute 驗證請求有效分鐘 Int 否 選擇性項目，本次身份驗證請求的驗證有效時間，
    以分為單位，預設為 10 分鐘，最小限制需 3 分鐘
    05 NotifyURL 驗證結果通知 URL String(100) 否 當AuthType=2 時，此欄位為必填
    Request Body Example

    {

    "AuthType" : "1" ,

    "ID" : "A00000000" ,

    "Birthday" : "19821108" ,

    "ExpireMinute" : "" ,

    "NotifyURL" : ""

    }
  • Response參數說明：
    Body
    No KEY KEY中文 型別 必填 說明
    01 RtnCode 回傳代碼 Int 是 回傳代碼
    錯誤代碼請參考狀態碼表
    02 RtnMsg 回傳訊息 String(100) 是 回傳訊息
    EX:訊息代碼：0001，成功
    其餘為失敗
    03 IsICPMember ICP會員 Int 是 是否為ICP會員
    1:是/0：否
    04 Timestamp 時間戳記 String(20) 是 格式：2019/01/01 00:00:00 UTC+8
    當 IsICPMember=1 時 ，回傳以下欄位
    05 Token 識別驗證碼 String(32) 是 本次身份驗證請求的識別驗證碼
    若請求的AuthType= 2時，請將Token轉為QR Code顯示於畫面，
    供ICPApp掃碼使用
    06 ExpireDate 驗證請求有效時間 Datetime 是 本次身份驗證請求的驗證有效時間
    (預設為 10 分鐘)
    Response Body Example

    {

    "RtnCode" : "0001" ,

    "RtnMsg" : "0001,成功" ,

    "IsICPMember" : "1" ,

    "Timestamp" : "2019/01/01 00:00:00 " ,

    "Token" : "Token000000000000000000000000001 " ,

    "ExpireDate" : "2019/01/01 00:10:00 "

    }

CHK002-身份驗證確認

特店於發送出身份驗證請求後：

1.若為APP推播驗證，則於使用者輸入驗證碼後呼叫此API進行ICP會員身份驗證確認
2.若為掃碼驗證，則僅需顯示QR Code於頁面上，無需呼叫此API

  • 介接路徑：
    UAT 環境： https://icash-apim.icashsys.com.tw/member-uat/v1/ICPMemberConfirm
    正式環境： https://icash-apim.icashsys.com.tw/member/v1/ICPMemberConfirn
  • Request參數說明：
    Header
    No KEY KEY中文 型別 必填 說明
    01 Ocp-Apim-Subscription-Key 存取權杖 String 是 由存取權杖中心配發，用以識別來源端是否合法
    Request Header Example

    {

    "Ocp-Apim-Subscription-Key" : "5f8b04501aa24e07b5adf96ed4b58203"

    }
    Body
    No KEY KEY中文 型別 必填 說明
    01 Token 識別驗證碼 String(32) 是 請求驗證確認的識別驗證碼
    02 AuthCode 使用者驗證碼 String(10) 是 使用者接收到的驗證碼(APP或簡訊)
    Request Body Example

    {

    "Token" : "Token000000000000000000000000001" ,

    "AuthCode" : "AC00000001"

    }
  • Response參數說明： tr>
    Body
    No KEY KEY中文 型別 必填 說明
    01 RtnCode 回傳代碼 Int 是 回傳代碼
    錯誤代碼請參考交易狀態代碼表
    02 RtnMsg 回傳訊息 String(100) 是 EX：訊息代碼：0001，成功
    其餘為失敗
    03 MemberAccount 會員電支帳號 String(20) 是 ICP電支帳號，168開頭
    04 MemberLevel 會員類型 Int 否 電支會員類型：
    Ex：1.一類會員/2.二類會員/3：三類會員
    05 IsAclBinding 是否綁定銀行直扣 Int 是 電支會員當前是否有綁定銀行直扣
    1：是/0：否
    06 IsCreditCardBinding 是否綁定信用卡扣款 Int 是 電支會員當前是否有綁定信用卡扣款
    1：是/0：否
    07 Timestamp 時間戳記 String(20) 是 格式：2019/01/01 00:00:00 UTC+8
    Response Body Example

    {

    "RtnCode" : "0001" ,

    "RtnMsg" : "0001，成功" ,

    "MemberAccount" : "16800000000000000001" ,

    "MemberLevel" : "2" ,

    "IsAclBinding" : "1" ,

    "IsCreditCardBinding" : "1" ,

    "Timestamp" : "2019/01/01 00:00:00 " ,

    }

CHK003-身份驗證結果通知

特店於發送出身份驗證請求後：

1.若為APP推播驗證，則使用者輸入驗證碼後呼叫CHK002進行ICP會員身份驗證確認。
2.若為掃碼驗證，則當使用者持ICP APP掃碼顯示於畫面上之QR Code進行身份驗證完成是，特店端於CHK001帶入的NotifyURL會收到ICP Server的Post通知驗證結果。

  • 介接路徑：
    {"特店提供"}
  • Request參數說明：
    Body
    No KEY KEY中文 型別 必填 說明
    01 Token 識別驗證碼 String(32) 是 請求驗證確認的識別驗證碼
    02 MemberAccount 會員電支帳號 String(20) 是 ICP電支帳號，168開頭
    03 MemberLevel 會員類型 Int 是 電支會員類型。
    EX：1：一類會員/2：二類會員/3：三類會員
    04 IsACLBinding 是否綁定銀行直扣 Int 是 電支會員當前是否有綁定銀行直扣
    1：是/0：否
    05 IsCreditCardBinding 是否綁定信用卡扣款 Int 是 電支會員當前是否有綁定信用卡扣款
    1：是/0：否
    Request Body Example

    {

    "Token" : "Token000000000000000000000000001" ,

    "MemberAccount" : "16800000000000000001" ,

    "MemberLevel" : "2" ,

    "IsACLBinding" : "1" ,

    "IsCreditCardBinding" : "1" ,

    }
  • Response參數說明：
    Body
    No KEY KEY中文 型別 必填 說明
    01 RtnCode 回傳代碼 Int 是 回傳代碼，1代表接收成功
    錯誤代碼請由特店端自行定義
    02 RtnMsg 回傳訊息 String(100) 是 回傳訊息
    EX：訊息代碼：0001，成功
    其餘為失敗
    03 Timestamp 時間戳記 String(20) 是 格式：2019/01/01 00:00:00 UTC+8
    Response Body Example

    {

    "RtnCode" : "1" ,

    "RtnMsg" : "0001，成功" ,

    "Timestamp" : "2019/01/01 00:00:00 " ,

    }

ICPOB000-建立授權扣款綁定要求

呼叫此API建立授權綁定要求。依據使用者載具不同，提供以下兩種綁定方式：

1.APP2APP或手機WEB2APP的授權綁定方式，特店發出建立授權扣款綁定要求，ICP中心段提供綁定授權碼，特店依據授權碼呼叫ICPOB002將ICP喚起進行授權綁定
2.PCWEB2APP的授權綁定方式，特店發出建立授權扣款綁定要求，ICP中心端提供綁定授權碼，特點於畫面顯示授權碼QR Code，由使用者手持ICP APP掃碼後進行授權綁定

  • 介接路徑：
    WEB版：
    正式環境： https://payment.icashpay.com.tw/api/V2/Payment/Binding/CreateICPBinding
    UAT 環境： https://icp-payment-preprod.icashpay.com.tw/api/V2/Payment/Binding/CreateICPBinding
  • Request參數說明：
    Header
    No KEY KEY中文 型別 必填 說明
    01 X-iCP-EncKeyID AES加密金要ID String 是
    02 X-iCP-Signature 用戶端簽章 String 是 ClientPrivateKey,SHA256withRSA
    對整個Body進行簽章
    Request Header Example

    {

    " X-iCP-EncKeyID" : "{EncKeyID}" ,

    "X-iCP-Signature" : "{Request body 簽章後的字串}" ,

    }
    Body(JSON内容)
    01 EncData AES加密資料 String(8) 是 加密資料（AES），JSON格式
    Request Body Example

    {

    "EncData" : "{對EncData内容加密後的字串}"

    }
    以下為EncData内容：
    01 PlatformID 平台編號 String(8) 是 平台編號（由愛金卡提供）
    平台商身份才需要帶入特定平台編號，如爲一般特店業者則帶入MerchantID（特店業者編號）
    02 MerchantID 特店業者編號 String(8) 是 特店業者編號（由愛金卡提供）
    03 BindingTradeNo 特店業者綁定序號 String(50) 是 特店業者綁定交易序號
    同個商家編號的特店業者綁定交易序號不可重複，可使用英數字
    04 StoreName 商店名稱 String(30) 是 綁定扣款之商店名稱
    05 BindingMode 綁定模式 Int 是 交易模式
    1：Token
        手機：APP2APP、EmbedAPP、手機WEB2APP
    2：QR Code
        PC掃碼綁定：PCWEB2APP
    06 CallbackURL 綁定通知回傳網址 String(250) 是 當消費者綁定完成後，會員綁定結果以Server端Post方式，回傳至此網址
    此網址單純作爲通知使用，不會進行導頁，導頁是由RedirectURL所對應。
    07 RedirectURL 綁定流程結束之導頁指定路勁 String(250) 否 選擇性項目，當APP綁定完成需要跳轉時代入此項。（若爲一般網頁路徑會以另開方式進行網頁開啓，一般網頁路徑限定爲Https，若為URL Scheme，則會直接跳轉）
    當ICP APP綁定流程結束後，無論同意授權與否，使用者點選返回，會導至此RedirectURL的指定路徑，並將綁定結果透過前述指定路徑，用一下2種方式傳送。（内容為ICPOB003内容，並做URLEncode）

    一般網頁路勁（限定Https）：QueryString
    URL Scheme：URLVariables

    EX： RedirectURL設定https://www.mypay.com/BindResult?orderid=xxx將會收到https://www.mypay.com/BindResult?orderid=xxx&TradeResultCode=0001...
    08 MerchantUserID 特店使用者ID String(20) 是 特店的會員識別ID，此值會於ICPOB002回傳，供特店識別對應使用。
    09 DisplayInformation 顯示資訊 String(250) 是 綁定時顯示的資訊文案説明。
    商品標的/扣款頻率/扣款金額/扣款金額上限。
    10 RedeemFlag 點抵金開關 Int 是 0：關閉 1：開啓
    用來設定此綁定訂單是否可以使用點數折抵。
    11 ExpiredType 綁定效期類型 Int 否 選擇性項目預設為1。
    1：長效期綁定/2：短效期綁定
    12 TotalAmtLimit 綁定扣款金額上限 String(12) 是 長效期：綁定訂單單次可請求之扣款金額之上限。
    短效期：綁定訂單之扣款金額上限。

    EX：10000（NT$100）
    （金額包含2位小數）等於15、16的加總
    13 NonPointAmt 不可贈點金額上限 String(12) 是 不可計算纍積贈點金額上限（金額包含2為小數）。若無則帶0

    EX：10000（NT$100）
    14 MaxMonthAmt 當月繳款上限 String(12) 是 當月使用者於ICP APP的交款上限，不可高於300,000（金額包含2位小數）。若無則帶0

    EX：1000000（NT$10000）
    RedeemFlag為1時下列值為必填
    15 ItemAmt 一般交易金額上限 String(12) 是 一般交易金額上限（包含2為小數）(因應手續費計算可有不同所區分)。若無則帶0

    EX：10000（NT$100）
    16 UtilityAmt 代收交易金額上限 String(12) 是 代收交易金額上限（包含2為小數）(因應手續費計算可有不同所區分)。若無則帶0

    EX：10000（NT$100）
    17 ItemNonRedeemAmt 一般交易不可折抵金額上限 String(12) 是 一般交易的不可折抵商品金額上限（包含2為小數）(因應手續費計算可有不同所區分)。若無則帶0

    EX：10000（NT$100）
    18 UtilityNonRedeemAmt 代收交易不可折抵金額上限 String(12) 是 代收交易的不可折抵商品金額上限（包含2為小數）(因應手續費計算可有不同所區分)。若無則帶0

    EX：10000（NT$100）
    ExpiredType為2時下列值為必填：
    19 ExpiredDate 綁定扣款有效期間 String(12) 是 選擇性項目，當綁定銀行為短效期時必填。
    綁定訂單之有效期限
